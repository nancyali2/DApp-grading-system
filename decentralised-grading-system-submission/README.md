# Decentralised Grading System тАФ Ethereum DApp

This project is a blockchain-based student grading system built on the Ethereum network. It was developed as part of **EE 465: Cryptocurrency and Blockchain Technology** at **IIT Bombay (Autumn 2019)**. The idea is simple: instead of relying on spreadsheets or databases that can be edited by anyone with access, all grading operations are handled by a smart contract on the Ethereum blockchain тАФ making every mark, weightage, and final grade transparent, permanent, and independently verifiable.

---

## What Problem Does This Solve?

In most universities, marks are entered into spreadsheets by teaching assistants and professors. Students have no way to verify that the calculations were done correctly, that a promised mark revision was actually applied, or that grades weren't changed after the fact. This system fixes all of that by putting every grading action on-chain:

- Every mark entry is a blockchain transaction тАФ it cannot be silently changed.
- Weighted totals and final grades are computed by the smart contract, not a formula in a spreadsheet.
- Any student can query their own marks, total, and grade directly from the blockchain at any time.

## How It Works

### Roles

The system has four roles, each with different permissions enforced by the smart contract:

- **Admin** тАФ the account that deployed the contract. Can register instructors.
- **Instructor (Professor)** тАФ can create courses, add exams, calculate final grades, and view all marks.
- **Teaching Assistant (TA)** тАФ can add exam marks and update existing marks.
- **Student** тАФ can only view their own marks and grade for a course they are enrolled in.

Role checks happen inside the contract using `require(msg.sender == ...)`. Since Ethereum transactions are signed with private keys, these checks are cryptographically enforced тАФ no username or password needed.

### Grading Workflow

1. Admin registers instructors by adding their Ethereum addresses.
2. Instructor creates a course тАФ specifies course ID, name, student roll numbers, student Ethereum addresses, and TA addresses.
3. After each exam, the Instructor or a TA calls `addExam` with the marks for each student.
4. If any marks need correction, `updateMarks` can be called тАФ this creates a new on-chain transaction, so the change is traceable.
5. Once all exams are done, the Instructor calls `calculateGrades` with the weightage for each exam and the grade cutoff thresholds.
6. The contract computes each student's weighted total and assigns a grade using the IIT Bombay grade chart (AA=10, AB=9, BB=8 ... F=0).
7. Students can then call `getStudentMarksGrades` to see their full breakdown.

### Grade Calculation

Weighted totals are computed using integer arithmetic with a precision multiplier of 1000 to avoid floating-point issues:

```
total += (marks * 1000 * weightage) / maxMarks
total /= 1000
```

This gives results equivalent to two decimal places of precision entirely within Solidity's integer math.

## Smart Contract: Grader.sol

The core of the project. Key functions:

| Function | Who Can Call | What It Does |
|----------|-------------|--------------|
| `addInstructor` | Admin | Registers instructor Ethereum addresses |
| `getInstructorsList` | Admin | Returns all registered instructors |
| `addCourse` | Instructor | Creates a course with students and TAs |
| `addExam` | Instructor, TA | Records marks for an exam |
| `updateMarks` | Instructor, TA | Updates marks for an existing exam |
| `calculateGrades` | Instructor | Applies weightages and computes final grades |
| `getProfExamMarks` | Instructor | Gets all student marks for one exam |
| `getProfExamWeightages` | Instructor | Gets exam list and weightages |
| `getProfMarksGrades` | Instructor | Gets totals and grades for all students |
| `getStudentMarksGrades` | Instructor, Student | Gets marks and grade for one student |
| `kill` | Admin | Destroys the contract |

## Installation and Setup

### Prerequisites

- Node.js (v10 or later)
- npm
- Truffle (`npm install -g truffle`)
- Ganache (download from https://trufflesuite.com/ganache/)
- MetaMask browser extension

### Step 1 тАФ Install dependencies

```bash
npm install
```

### Step 2 тАФ Start Ganache

Open Ganache and start a new workspace. Note the RPC server address (usually `http://127.0.0.1:7545`). Check that the port in `truffle-config.js` matches:

```js
development: {
  host: "127.0.0.1",
  port: 8545,       // change this to match your Ganache port
  network_id: "*",
}
```

### Step 3 тАФ Connect MetaMask

In MetaMask, add a custom network pointing to the same RPC URL as Ganache. Then import one of the Ganache accounts using its private key тАФ this account will act as the Admin.

### Step 4 тАФ Compile and deploy the contract

```bash
truffle compile
truffle migrate
```

Truffle will print the deployed contract address. The `build/contracts/Grader.json` file gets updated automatically and the front end reads from it.

### Step 5 тАФ Run the DApp

```bash
npm run dev
```

This starts `lite-server` and opens `http://localhost:3000` in your browser. The DApp is now live and connected to your local Ganache blockchain.

## Using the DApp

The UI has two sidebars тАФ select your role on the left (Admin, Professor, TA, Student) and a function panel appears on the right. Click any function to open the form in the centre. Fill in the fields and submit. MetaMask will pop up asking you to confirm the transaction.

**Example: Adding a course**
- Course ID: `EE465`
- Course name: `Blockchain Technology`
- Roll list: `190010001,190010002,190010003`
- Student addresses: `0xabc...,0xdef...,0x123...`
- TA addresses: `0x456...`

**Example: Calculating grades**
- Course ID: `EE465`
- Weightages: `10,30,60` (quiz, midsem, endsem тАФ must sum to 100)
- Grade cutoffs: `85,75,65,55,45,35,25` (for AA, AB, BB, BC, CC, CD, DD)

## Known Limitations

- **No privacy between students** тАФ a student who knows another student's roll number could theoretically query their marks by calling the contract directly (bypassing the UI). Ring signatures or zero-knowledge proofs would fix this.
- **Gas cost** тАФ storing large arrays of marks on-chain is gas-intensive for big classes.
- **Solidity 0.4.26** тАФ this version lacks built-in overflow protection. For production, upgrading to 0.8.x is recommended.
- **No CSV upload** тАФ marks have to be typed in as comma-separated values.
